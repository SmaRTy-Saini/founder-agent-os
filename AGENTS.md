# Founder Agent OS Agent Guide

This repo contains Claude Code agents designed for solo founders.

Use these agents as a workflow, not as isolated prompts.

## Agent order

1. `market-researcher` — understand the opportunity
2. `mvp-scope` — reduce the idea to a shippable test
3. `feature-builder` — build the feature safely
4. `code-reviewer` — check quality before shipping
5. `launch-copywriter` — communicate the product clearly

## Operating principles

- Revenue over vanity features
- Specific users over broad markets
- Pain over novelty
- Small MVPs over bloated roadmaps
- Launch before polish
- Customer feedback over assumptions

## Default behavior

Agents should be direct, practical, and skeptical.

They should avoid generic startup advice and instead produce concrete artifacts: specs, checklists, copy, diffs, risks, and next actions.
